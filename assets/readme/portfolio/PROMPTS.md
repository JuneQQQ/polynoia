# README hero image prompts

All three images were generated with OpenAI ImageGen in built-in generation
mode, then converted to repository-local JPEG files. They are conceptual
illustrations—not product UI screenshots.

## `hero-global-oss.jpg`

> Create a wide 16:9 editorial hero illustration for an engineering project
> README. Show multiple isolated streams of work—abstract code fragments,
> branching paths, and small glowing nodes—converging through a precise verified
> merge gate into one stable mainline. Dark navy and graphite background, cool
> cyan with restrained amber accents, sophisticated technical magazine art,
> strong depth and clean negative space, crisp geometric forms, subtle paper and
> glass texture. The idea is reliable multi-agent coordination and deterministic
> integration. No people, no faces, no robots, no laptop, no browser window, no
> application UI, no dashboard, no screenshots, no logos, no readable text, no
> letters, no numbers, no watermark.

## `hero-campus-cn.jpg`

> Create a wide 16:9 conceptual cover illustration for a Chinese campus
> recruiting software-engineering portfolio. On an elegant warm ivory paper
> field, depict several initially tangled distributed paths becoming carefully
> separated work lanes, passing through test checkpoints, and converging into
> one clean dependable result. Use hand-crafted technical diagram energy,
> precise ink lines, subtle cobalt blue, vermilion, charcoal and muted gold,
> tasteful editorial design, confident but not flashy, generous negative space.
> Communicate that a student engineer turned an AI demo into a dependable
> system. No people, no mascots, no product screens, no browser chrome, no UI
> cards, no fake screenshot, no logos, no readable text, no letters, no numbers,
> no watermark.

## `hero-case-study.jpg`

> Create a wide 16:9 forensic engineering case-study illustration. Visualize a
> system timeline as layered translucent strata: message events, receipts,
> retries, reconnects, database commits, isolated worktrees, and a merge boundary
> represented only through abstract marks and connected geometry. A few red
> anomaly traces are intercepted and resolved into a calm green-blue terminal
> state. Dark slate background, precise cyan lines, small amber and red signals,
> architectural blueprint plus observability-trace aesthetic, cinematic yet
> rigorous, highly legible composition with open space for a README heading
> above or below. No product interface, no dashboard, no terminal screenshot, no
> browser, no people, no logos, no readable text, no letters, no numbers, no
> watermark.
