① 一份 prompt = 五层叠起来

用户敲完回车后,build_context_for_turn() 把下面五块拼起来,作为最终 prompt 喂给子进程:

L1
身份
"你是 @claudeCode,在 Polynoia 上担任..."
来源:agents 表 · identity.py
2,000 tok
L2
项目简介
"当前在 #api-redesign,你也是 #billing / #infra 的成员"
来源:workspaces 表 · briefs.py
3,000 tok
L3
跨 conv 活动账本 ★ 关键
"3 天前在 #conv-tax-fix,用户让你修 round 调用,你改了 line 42"
来源:messages 表跨 conv 查 · ledger.py(30 条 / 每群 8 条)
15,000 tok
L4
本 conv 历史
当前对话最近 30 条消息
来源:messages 表本 conv 查 · history.py
35,000 tok
L5
当前用户消息
用户这一 turn 输入的原文
来源:本次 WebSocket payload
5,000 tok
window.py · 超 60k tok 就从老的开始截
最终 prompt 字符串
用 \n\n---\n\n 拼接 5 层 → 喂给 adapter 的 send(text=...)
L3 才是让 Agent "像同事"的那一层。 之前没这层时,agent 每次见你都失忆; 现在它能说"上周你让我做 X,我交付了 Y" —— 不是真记忆,是每次说话前用 SQL 查出来重新喂。

② 一个真实例子:模型实际收到的 prompt

L1
# 身份 你是 @claudeCode,Polynoia 上的 Anthropic 代码 Agent。 擅长 React / TypeScript / 重构。
L2
# 你的项目 当前对话所在项目:#api-redesign 你也是这些项目的成员:#billing, #infra
L3
# 你的近期活动 ## #billing · #conv-tax-fix (dm) - 3 天前 用户: 帮我看 calculator.py 的 tax 计算 - 3 天前 @claudeCode(你): [工具调用 edit/completed] 修 line 42 - 3 天前 @claudeCode(你): 改完了,改成 ROUND_HALF_EVEN ## #api-redesign · #conv-schema (group) - 昨天 用户: 设计一下 invoice 的 schema - 昨天 @claudeCode(你): 我建了 5 个 Pydantic model: Invoice / LineItem / ...
L4
# 本对话历史(最近 30 条) [当前 conv 的近期消息,本例略]
L5
# 当前用户消息 帮我加一下 invoice 的折扣逻辑
模型看到这一坨之后心里:"哦是 billing.py 的 Invoice,昨天刚跟用户聊过 schema" → 直接在 LineItem 旁边加 discount 字段 + 沿用 tax 的 config.toml 模式 → 用户感觉"它真的记得我们聊过的东西"。

④ 还缺什么

缺 Orchestrator 还在用老路径

当前 sub-agent 直接路径有 5 层,跑 orchestrator 路径反而零 context。行为分裂。

缺 Prompt cache 标记

L1+L2 几乎不变,但没打 cache_control 标记,每 turn 重发烧钱。

半 Git commit 集成到 L3

ledger.py:6-8 注释里规划了,但代码只查 messages 表,git log 没接。

缺 LLM 摘要压缩(P1)

超 token 预算时只截断,不压缩。老消息直接没了,不像人脑会留下印象。

半 真 tokenizer

现在用 len(text)//3 估算 token 数,中英混合边界 case 会偏。

缺 语义检索(spec 的 L5)

没接 sqlite-vec,远期模糊召回("4 周前说过类似的事")做不到。

⑤ 一句话评价

上次讨论里提的 episodic memory(L3 跨 conv 账本)和 Claude Code 的 append 模式都落地了,而且做得很干净: context/ 独立成包,有测试覆盖关键 invariant(同 adapter 多 contact 账本不串)。 后面接 Orchestrator + Prompt cache + Git log 这三件,会让产品质感再上一档。
