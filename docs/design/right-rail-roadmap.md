# 右侧栏(代码 / 预览区)演进 Roadmap

> 右侧栏 = 聊天区右边那块「代码 / 预览」面板。这里记录它的现状、后续规划、以及关键的方向调整,供团队对齐 + 答辩。

## 现状(2026-06-01)

右侧栏顶部「代码 / 预览」切换,按场景渲染:

- **代码**:CodeMirror 编辑任意文件(VSCode 键位 / 查找替换 / 小地图),`Ctrl+S` → PUT → 自动 commit;文件落 main 后**文件树自动刷新 + 自动打开入口**(无需手动刷新)。
- **预览**(按文件类型分派):
  - `.md` 文档 → **milkdown crepe 所见即所得编辑**(居中文档版式),保存写回。
  - Marp `.md`(`marp: true`) → **翻页幻灯**实时渲染。
  - `.csv` → **Excel 式表格**(SheetJS)。
  - `.html` → **iframe** 静态页面预览。
- **导出**:文档→PDF/.md/.docx,PPT→PDF/.pptx(纯文本),表格→.xlsx/.csv/PDF,网页→PDF/.html。
- **冲突解决**面板(冲突闭环)、**改动评审**面板(manual 合并)。

定位:**轻量、可移植、真实时**——agent 产出文本(md/csv/Marp/html),前端渲染成 Office 风格 + 导出,不依赖宿主环境。

## 后续规划

### 右侧终端显示(配合大型项目)
- **背景**:大型/重型项目技术栈与环境千奇百怪,"把整个项目实时渲染成一个页面"既重又不现实(见下「历史决策」)。对这类项目,开发者真正需要的是一个**终端**。
- **目标**:在右侧栏增加**终端**面板,可看命令输出 / dev server 日志 / 构建过程,并(进一步)交互式跑命令。
- **配合**:与「代码 / 预览」并列的第三种右栏模式;大项目用终端,轻产出(文档/PPT/表格/静态页)用预览。
- **待设计**:终端跑在哪(workspace worktree 的 shell)、安全边界(沙箱 §6.2,P0 无强隔离)、与 agent 工具 `run_shell` 的关系、输出流式推送(复用现有 WS/SSE 通道)。
- **状态**:规划中,当前先把轻量预览(文档/PPT/表格/静态页)做好。

## 历史决策

### Docker 项目运行器(2026-05-31 实现 → 2026-06-01 回滚)
- **曾做**:检测 workspace 项目类型(static / npm / python)→ 在 Docker 容器里把**整个项目跑起来** → iframe 指向它预览;含 import 推断装依赖、sitecustomize 强制 host/port、反向/直连端口。
- **回滚原因**:
  1. **太重**:每次起容器 + 装依赖几十秒,谈不上"实时"。
  2. **环境杂**:大项目技术栈/版本各异,容器未必装得对、跑得起。
  3. **方向错配**:用户要的"实时渲染"其实是文档/PPT/表格这类高频产出 → 改前端渲染;大项目交互 → 改「右侧终端」(上)。
- **结论**:`runners/` 整套删除,依赖切换(去 @babel/standalone、加 milkdown/marp/xlsx/pptxgenjs/html-docx)。测试案例存档见 `docs/testing/manual-test-cases.md` 文末。
