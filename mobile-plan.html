Polynoia · 移动端实现规划

iOS / Android — web/桌面端的轻量子集 · 实现方案 v1

把现有 web 应用装进原生壳里运行,适配移动布局,做成 rule.md 定义的「轻量 IM:查看对话、审批确认、产物预览」—— 不重写、不偏离。

架构:Capacitor 6 复用 web 范围:IM + 审批 + 只读预览 复用 ~85% 客户端逻辑 7 个阶段
为什么这么做

架构决策:Capacitor 复用 web,而非 React Native 重写

apps/mobile/ 已经是一个 Capacitor 6 脚手架(webDir: "../web/dist",已配 keyboard / network / preferences / splash / status-bar 插件)。移动 App = 同一套 apps/web React 代码跑在原生 WebView 里,通过 apps/web/src/lib/platform.ts 的 isMobile() 自适应成移动布局(App.tsx 已有「抽屉 + 单列」分支)。

为什么不选 React Native: web 客户端约 85% 是纯 TS(store.ts / lib/api.ts / lib/ws.ts / lib/types.ts),已经运行时自适应。Capacitor 直接复用这套 UI 与流式/WS/store 逻辑 —— 最贴合「子集、不偏离」,远比 RN + packages/ui-rn 重写轻。(注:CLAUDE.md §6.3 旧文写的是 RN,需更新为 Capacitor;附 ADR。)

  • lib/runtime-config.ts — 服务器基址单一来源:web 为相对路径 "";移动端为持久化的绝对 URL;wsBase() 推导 ws://。
  • lib/storage.ts — 持久化抽象:web 用 localStorage,原生动态 import @capacitor/preferences;带同步缓存 + prefetchStorage()。

服务器连接层(上设备的关键 —— WebView 不能用相对 URL / window.location)

  • api.ts + ws.ts 全部走 serverBase() / wsBase()。
  • 新增 mobile/ConnectServerScreen.tsx:输入局域网 IP → 测 /api/health → 持久化;移动端首屏 gate。Web 路径不变。

注意

风险登记

WebView CORS Origin 不确定 —— https://localhost 还是 capacitor://localhost 取决于 Capacitor 版本与 scheme;上线前从服务端日志实测确认。allow_credentials=True 禁止通配,必须显式列表。
异步 Preferences 启动闪烁 —— 用 prefetchStorage 同步缓存 + SplashScreen 覆盖。主题留 WebView localStorage 供预绘制脚本。
WS 后台/前台重连 —— App 挂起时 socket 静默断;靠 @capacitor/app resume + Network online 触发 reconnect() + 状态重同步,否则状态 pill 会卡「运行中」。
混合内容(https WebView → http 局域网服务器) —— 可能被拦;allowNavigation 放行 + 上线建议走 https/*.polynoia.app;连接页明确报错。
