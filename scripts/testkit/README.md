# testkit

官方提交与演示只保留一个初始化入口:

```bash
bash scripts/testkit/reset.sh
```

这个脚本会完成完整准备流程:

1. 停掉本机旧的 Polynoia 后端/前端开发服务。
2. 清空本地数据库并重建 schema。
3. 启动后端 `:7780` 与前端 `:5173`。
4. 灌入一组真实可交付项目用例,每个用例对应独立项目/工作区。

默认用例覆盖:

- 网页小游戏:飞机大战、斗地主、2048。
- 办公产物:PPT 路演稿、DOCX 合同、Excel 家庭预算。
- 网站与内容:咖啡店官网、摄影师作品集、英语夏令营落地页、旅行计划。
- 数据分析:电商销售分析报告。
- 协作边界:多人接力、未知 @、同文件冲突、单 Agent main 同步、Diff/历史、缺失资料恢复。

设计原则:

- 不再维护多套 demo seed 脚本。新增官方演示/回归案例时,直接编辑
  `reset.sh` 内的 `CASES` 列表,保证一条命令能复现完整演示状态。
- 不把所有案例塞进同一个“项目库”。每个案例都是独立真实项目,便于测试项目切换、
  成员范围、归档、历史、Diff 和 main 同步。
