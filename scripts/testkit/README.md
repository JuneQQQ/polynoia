# testkit

官方提交与演示只保留一个初始化入口:

```bash
bash scripts/testkit/reset.sh
```

这个脚本会完成完整准备流程:

1. 停掉本机旧的 Polynoia 后端/前端开发服务。
2. 清空本地数据库并重建 schema。
3. 启动后端 `:7780` 与前端 `:5173`。
4. 灌入一组贴近线上上线准备的真实用例。

默认用例覆盖:

- 发布页、Release Notes、QA 检查表、状态页、埋点验收报告。
- Go-live 多 Agent 协作包。
- @ 路由:顺序依赖、单点名直达、未知成员忽略。
- 合并与协作边界:同文件冲突、单聊 main 同步、Diff/历史、工具错误恢复。

设计原则:不要再维护多套 demo seed 脚本。新增官方演示/回归案例时,直接编辑
`reset.sh` 内的 `CASES` 列表,保证一条命令能复现完整演示状态。
